# 强化学习简明笔记

## 1. 什么是强化学习？

强化学习（Reinforcement Learning，RL）是机器学习的一个分支，它关注**智能体（Agent）**如何在一个**环境（Environment）**中通过**试错**来学习，从而获得最大的**累积奖励**。

与有监督学习不同，强化学习没有“正确答案”标签，只有“好”或“坏”的反馈信号（奖励）。与无监督学习不同，强化学习的目标是最大化累积奖励，而不是发现隐藏结构。

**核心关键词**：试错、延迟反馈、序列决策。

---

## 2. 基本组成要素

一个标准的强化学习问题包含以下部分：

| 要素 | 含义 | 示例（围棋） |
|------|------|--------------|
| Agent | 学习并做决策的主体 | 下棋的程序 |
| Environment | Agent 交互的外部系统 | 棋盘、对手 |
| State (s) | 环境的当前状况 | 当前棋盘布局 |
| Action (a) | Agent 在某个状态下可以做的操作 | 落子的位置 |
| Reward (r) | 执行动作后环境返回的即时得分 | 吃子得1分，输棋得-1分 |
| Policy (π) | Agent 根据状态选择动作的策略 | 决定在哪里落子的规则 |
| Value function | 评估某个状态或动作的“长期好程度” | 某步棋的胜率预测 |

---

## 3. 核心概念解释

### 3.1 环境与智能体的交互循环

1. Agent 观察当前状态 \( s_t \)
2. Agent 根据策略选择动作 \( a_t \)
3. 环境执行动作，转移到新状态 \( s_{t+1} \)
4. 环境返回奖励 \( r_{t+1} \)
5. Agent 利用 \((s_t, a_t, r_{t+1}, s_{t+1})\) 更新策略
6. 重复以上步骤

### 3.2 奖励与回报

- **即时奖励** \( R_t \)：一步动作后的反馈。
- **累积回报** \( G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \dots \)
- **折扣因子** \( \gamma \in [0,1] \)：让未来的奖励“打折”，体现短期优先或环境不确定性。

### 3.3 策略（Policy）

- **确定性策略**：\( \pi(s) = a \)，给定状态输出唯一动作。
- **随机性策略**：\( \pi(a|s) = P(A=a|S=s) \)，输出动作的概率分布。

### 3.4 值函数（Value Function）

- **状态价值函数** \( V^\pi(s) \)：在状态 \( s \) 下，遵循策略 \( \pi \) 所能获得的期望累积回报。
- **动作价值函数** \( Q^\pi(s,a) \)：在状态 \( s \) 下采取动作 \( a \)，之后遵循策略 \( \pi \) 的期望回报。

---

## 4. 主要算法分类

### 4.1 基于值函数的方法（Value-based）

- 核心思想：学习最优的 \( Q \) 函数，然后从 \( Q \) 中导出策略 \( \pi(s) = \arg\max_a Q(s,a) \)
- 经典算法：**Q-learning**、**DQN**（深度 Q 网络）

### 4.2 基于策略的方法（Policy-based）

- 核心思想：直接学习策略 \( \pi(a|s) \)，通常用神经网络表示
- 优势：适合连续动作空间、收敛性更好
- 经典算法：**REINFORCE**、**PPO**、**TRPO**

### 4.3 演员-评论家方法（Actor-Critic）

- 结合两者：Actor 学习策略，Critic 评估当前策略的值函数
- 经典算法：**A2C**、**A3C**、**SAC**

---

## 5. 关键挑战与概念

- **探索 vs 利用**：Agent 既要尝试新动作（探索）以获得更好策略，也要执行已知好动作（利用）以获得奖励。
- **延迟奖励**：可能很多步之后才有奖励，需要信用分配（Credit Assignment）。
- **马尔可夫决策过程（MDP）**：强化学习常用的数学框架，假设未来只与当前状态有关。
- **部分可观测环境（POMDP）**：Agent 看不到完整状态，需要记忆或推测。
